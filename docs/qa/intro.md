# QA Guides

Quality is not a phase at the end — it's woven through the lifecycle. These guides cover how QA collaborates with devs, plans testing, automates well, files actionable bugs, and prevents regressions.

## What's inside

| Page | What you'll learn |
|---|---|
| [Test Strategy](./test-strategy) | The test pyramid, what to automate vs. exploratory |
| [Test Automation](./automation) | Frameworks (Playwright, Cypress, Selenium), patterns |
| [Bug Reporting](./bug-reporting) | The anatomy of a bug report that gets fixed fast |
| [Regression](./regression) | How to keep old bugs from coming back |
| [Quiz](./quiz) | Self-assessment |

## QA is engineering

The historic divide between "developers" and "QA" is unhelpful. The best QA engineers code, the best developers care about test quality. Both audiences should read both this and the [Dev Guides](../dev/intro).
